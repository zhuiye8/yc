# 前端数据来源说明

> 更新日期: 2026-03-29
> 说明: 以下整理了各页面模块的数据来源情况，标注了哪些是接口真实数据、哪些是本地数据或按比例估算的数据。

---

## 一、产业招引

| 模块 | 数据来源 | 说明 |
|------|---------|------|
| 产业链图谱（树形结构） | 本地数据 | 6条产业链的上中下游树形结构由前端本地维护（MD文件），非接口 |
| 节点浮窗（企业数/人才数） | 接口：findOrgByModelsDecode-v1 + findExpert-v2 | 并行4个请求（全国/本地各一次），真实数据 |
| 链上企业列表 | 接口：findOrgByModelsDecode-v1 | 真实数据，使用宽关键词搜索 |
| 链上人才列表 | 接口：findExpert-v2 | 真实数据，使用宽关键词搜索 |
| 本地化率 | 接口计算 | 本地企业数 ÷ 全国企业数，实时计算 |
| 预警系数 | 假数据 | 固定值72.5，需客户提供算法或接口 |
| 搜索框 | 接口：findOrgByModelsDecode-v1 + findExpert-v2 | 真实数据 |

## 二、人才引育

| 模块 | 数据来源 | 说明 |
|------|---------|------|
| 人才搜索 | 接口：findExpert-v2 | 真实数据 |
| 人才关系图谱 | 接口：graphSinglePersonRelation | 真实数据，部分人才可能无合作关系（单节点显示） |
| 当前人才卡片 | 接口：findExpert-v2 + getExpertSummary | 真实数据（姓名/机构/H指数/论文/专利/成果） |
| 合作人才列表 | 接口：talent-coopTalentList | 真实数据，但该接口偶尔502不稳定 |
| 高端人才榜 | 接口：findExpert-v2 | 搜索结果按H指数排序取前6名，真实数据 |
| 研究方向分布（柱状图） | 接口：findExpert-v2 | 从搜索结果的KEYWORDS字段聚合统计，真实数据 |
| 研究方向趋势（折线图） | 接口：getCkeyIndustry | 真实数据（论文/专利/标准年度趋势） |
| 宜昌籍人才列表 | 本地数据 | 客户提供的Excel（24,970条宜昌技能人才），转为JSON加载 |

## 三、大屏 - 产业布局

| 模块 | 数据来源 | 说明 |
|------|---------|------|
| 6条产业链选择 | 本地数据 | 产业链名称固定 |
| 产业链综合指标 - 缺链数/强链数 | 假数据 | 固定值4和10，需客户提供接口 |
| 产业链综合指标 - 产业链数 | 固定值 | 固定为6 |
| 产业链综合指标 - 企业总数 | 接口：findOrgByModelsDecode-v1 | 真实数据（接口返回的total） |
| 产业链综合指标 - 人才总数 | 接口：findExpert-v2 | 真实数据（接口返回的total） |
| 链上企业列表 | 接口：findOrgByModelsDecode-v1 | 真实数据 |
| 链上人才列表 | 接口：findExpert-v2 | 真实数据 |
| 企业类型分布（柱+线图） | 接口 + 比例估算 | 从接口返回的200条企业样本中统计TAGS（上市/专精特新/科技型中小），按比例推算到总数。全量科技=接口total原值 |
| 中央地图 | 接口：getCkeyMap | 真实数据（各省数量着色） |
| 全国/宜昌视图切换 | 接口联动 | 切换宜昌聚焦时，链上企业和人才加city=宜昌重新请求 |

## 四、大屏 - 人才总览

| 模块 | 数据来源 | 说明 |
|------|---------|------|
| 全国人才分布（左上柱状图） | 接口：getCkeyMap('人才') | 真实数据，取Top11省份 |
| 人才分类（左中玫瑰图） | 接口 + 固定比例 | 总数来自getCkeyMap各省加总，按6:3:1分配为创新/技能/领军，比例为估算值 |
| 人才总数（左下大数字） | 接口：getCkeyMap('人才') | 各省加总，真实数据 |
| 人才等级分布（左下横向柱状图） | 固定比例 | 基于总数按比例估算：活跃28%、重点6%、院士/顶尖0.1%，需客户提供精确接口 |
| 宜昌籍人才分布（右上柱状图） | 接口：getCkeyMap('宜昌') | 近似数据，搜索"宜昌"关键词≠宜昌籍贯，需客户提供精确接口 |
| 宜昌人才所属产业链分布（右中环形图） | 接口：findExpert-v2 × 6次 | 每条产业链用宽关键词+city=宜昌查询total，真实数据 |
| 宜昌人才缺口数（右下柱+线图） | 本地数据 | 客户提供的岗位表Excel（近12个月188条岗位需求），按岗位名映射到6条产业链统计 |

## 七、补充说明：产业链别名问题

**现象**：6条产业链中，部分产业链直接用原名搜索在万方数据库中匹配度很低或为0。

| 产业链 | 原名搜索企业数 | 宽关键词搜索企业数 |
|--------|-------------|----------------|
| 湿电子化学品 | 29 | 237（电子化学品 OR 半导体材料 OR 湿电子） |
| 新能源新材料 | 22 | 23,080（新能源 OR 新材料 OR 电池 OR 储能） |
| 先进制剂与高端仿制药 | **0** | 4,546（制药 OR 仿制药 OR 生物医药 OR 药物制剂） |
| 酵母发酵与功能成分制造 | **0** | 5,497（酵母 OR 发酵 OR 生物工程 OR 功能食品） |
| 内河绿色智能船舶制造 | **0** | 8,702（船舶 OR 造船 OR 航运 OR 船舶制造） |
| 人工智能 | 15,675 | 15,675（无需别名） |

**处理方式**：前端为每条产业链配置了"宽关键词别名"，用OR关联多个相关词搜索，确保所有产业链都能返回数据。此方案为临时措施，建议后续由数据侧优化产业链与万方数据的映射关系。

### 1.2 需要甲方提供的内容

| 序号 | 内容 | 说明 |
|------|------|------|
| 1 | **完整接口文档**（Swagger/OpenAPI 或同等） | 包含所有接口的 URL、请求方法、请求参数、响应结构、字段含义 |
| 2 | **VPN 接入凭证与指引** | VPN 客户端、账号密码、内网域名/IP、端口信息 |

| 5 | **测试账号** | 带有完整数据权限的测试账号 |
| 6 | **数据字典** | 所有枚举值、状态码、行政区划编码等标准代码表 |
| 7 | **接口变更通知机制** | 接口更新后如何通知前端（文档版本管理 or 群通知等） |

### 1.3 前端技术栈信息

- 框架：React 19 + TypeScript + Vite
- HTTP 库：建议使用 Axios（当前项目未引入，接口对接时会新增）
- 图表库：ECharts 6.0（echarts-for-react）
- 地图：ECharts 地图组件（需 GeoJSON 数据，当前使用本地 `china-map.json`）

### 2.1 首页（Home）

#### 接口 1：五大模块总览统计

> 首页核心展示区域，五张卡片分别展示产业/人才/创新/资金/政策的核心KPI指标。

**请求参数：**
- `region`：区域筛选（省→市→区县三级，如 `["hubei", "yichang", "xiling"]`）
- `timeRange`：时间窗（枚举值：`1y` / `3y` / `5y` / 自定义起止日期）
- `industryChain`：产业链筛选（如 `["chemical", "chem-phosphorus"]`，不选则为全部）

**需要返回的字段：**

### 2.2 产业模块（Industry）

#### 接口 4：产业链图谱数据

> 核心功能。按产业链展示上中下游树形结构，每个节点标注强链/弱链/缺链状态。

**请求参数：**
- `chainKey`：产业链标识（`chemical` / `energy` / `bio` / `auto` / `bigdata`）

**需要返回的字段（树形结构）：**

```json
{
  "upstream": {
    "label": "上游",
    "root": {
      "name": "绿色化工",
      "status": "strong",         // 枚举：strong(强链) / weak(弱链) / missing(缺链)
      "enterprises": 189,         // 全国企业数
      "talents": 320,             // 全国人才数
      "localEnterprises": 85,     // 宜昌本地企业数
      "localTalents": 150,        // 宜昌本地人才数
      "children": [
        {
          "name": "化工上游原料",
          "status": "strong",
          "enterprises": 68,
          "talents": 120,
          "localEnterprises": 35,
          "localTalents": 60,
          "children": [
            {
              "name": "硫酸",
              "status": "strong",
              "enterprises": 28,
              "talents": 40,
              "localEnterprises": 15,
              "localTalents": 20
            }
          ]
        }
      ]
    }
  },
  "midstream": { /* 同上结构 */ },
  "downstream": { /* 同上结构 */ }
}
```

> **说明**：前端需要每个节点的 `status` 字段来着色（强链=蓝色/弱链=浅蓝/缺链=灰色），以及企业数和人才数用于弹窗展示。树形结构支持任意层级嵌套。

#### 接口 5：产业总览统计

> 选中某条产业链后，展示该链的总览数据，包含企业层级金字塔、省份分布、人才分布、类型占比。

**请求参数：**
- `chainKey`：产业链标识

**需要返回的字段：**

```json
{
  "summary": {
    "chainCount": 8,              // 环节总数
    "strong": 10,                 // 强链环节数
    "weak": 7,                    // 弱链环节数
    "missing": 6,                 // 缺链环节数
    "enterprises": 1890,          // 该链企业总数
    "talents": 3200               // 该链人才总数
  },
  "institutionType": [
    { "name": "上市公司", "value": 12 },
    { "name": "专精特新企业", "value": 86 },
    { "name": "科技型中小企业", "value": 420 },
    { "name": "全量科技企业", "value": 1890 }
  ],
  "provinceEnterprise": [
    { "name": "湖北", "value": 580 },
    { "name": "江苏", "value": 320 }
  ],
  "provinceTalent": [
    { "name": "湖北", "value": 480 },
    { "name": "江苏", "value": 356 }
  ],
  "talentType": [
    { "name": "院士", "value": 12 },
    { "name": "领军人才", "value": 98 },
    { "name": "骨干人才", "value": 856 },
    { "name": "基础人才", "value": 2234 }
  ]
}
```

> **说明**：
> - `institutionType` 用于3D金字塔图（从少到多，上市→专精特新→科技型中小→全量）
> - `provinceEnterprise` 用于3D柱状图（Top 8省份）
> - `provinceTalent` 用于中国地图热力渲染（需覆盖全国31个省级行政区）。省份名使用简称（如"湖北"、"内蒙古"），前端有映射表自动转全称
> - `talentType` 用于环形图（颜色由前端固定分配，无需后端返回）

### 3.1 VPN 接入需求

请提供以下信息：

| 项目 | 说明 |

| API 文档地址 | Swagger UI 或其他文档地址 |
| 可用时段 | 内网服务的可用时间段（是否 7×24 在线） |

### 3.2 前端本地开发联调方式

#### 方案一：Vite 开发代理（推荐）

连接 VPN 后，在 `vite.config.ts` 中配置代理，将 API 请求转发到内网服务器：

```typescript
// vite.config.ts
export default defineConfig({
  plugins: [react()],
  server: {
    port: 5175,
    proxy: {
      '/api': {
        target: 'http://内网API地址:端口',
        changeOrigin: true,
        // 如果接口路径不以 /api 开头，可以重写：
        // rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
});
```

前端代码中统一使用 `/api/xxx` 路径请求，由 Vite 代理转发到内网。

#### 方案二：环境变量切换

```bash
# .env.development（本地开发，走代理）
VITE_API_BASE_URL=/api

# .env.production（生产部署）
VITE_API_BASE_URL=http://实际生产API地址
```

前端代码中：
```typescript
const API_BASE = import.meta.env.VITE_API_BASE_URL;
```

#### 方案三：Mock → 真实接口渐进切换

在对接初期，可以按模块逐步切换。保留 Mock 数据作为 fallback，当 API 不可用时自动降级：

```typescript
async function fetchWithFallback<T>(url: string, mockData: T): Promise<T> {
  try {
    const res = await axios.get(url);
    return res.data;
  } catch {
    console.warn(`API ${url} 不可用，使用 Mock 数据`);
    return mockData;
  }
}
```

### 3.3 需要确认的网络环境问题

1. **VPN 连接后是否全局代理？** 还是仅内网网段走 VPN（Split Tunnel）？
2. **是否有 IP 白名单限制？** 开发人员的 IP 是否需要加白？
3. **内网 DNS 解析**：内网域名是否需要手动配置 hosts？
4. **证书问题**：内网 HTTPS 是否使用自签证书？如果是，需要提供根证书。
5. **防火墙端口**：前端开发端口（5175）是否需要开放？跨域（CORS）是否已配置？

```typescript
// src/services/api.ts
import axios from 'axios';

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// 请求拦截器 - 添加 Token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器 - 统一错误处理
request.interceptors.response.use(
  (response) => response.data,
  (error) => {
    if (error.response?.status === 401) {
      // Token 过期，跳转登录
    }
    return Promise.reject(error);
  }
);

export default request;
```
